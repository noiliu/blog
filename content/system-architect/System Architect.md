UDDl(Universal Description，Discovery&Integration)，UDDl用于Web服务注册和服务查找;
WSDL(Web Service Description Language)，对服务进行描述的语言，用于描述Web服务的接囗和操作功能;
SOAP(Simple ObiectAccess Protocol)，用于实现web服务的远程调用，定义了服务请求者和服务之间的消息传输规范（PRC），为建立Web服务和服务请求之间的通信提供支持。
BPEL(Business Process Execution Language For Web Services)，通过组合、编排和协调 Web 服务自上而下地实现面向服务的体系结构(SOA)，面向Web 服务的业务流程执行语言，它是一种使用 Web 服务定义和执行业务流程的语言。BPEL 提供了一种相对简单易懂的方法，可将多个 Web 服务组合到一个新的复合服务(称作业务流程)中。

REST中的“状态"分为应用状态(客户端保存)和资源状态(服务端保存);“表述”指的是资源在某一时间点的状态快照(表现
形式常见如JSON/XML)

n个节点，用n-1条边把所有的节点都联通，且不可生成环路：在图中依次找出最短的边，把他连起来，先把没有边的节点画出来，以次取最短的边连起来，不产出环路即可

  
![Pasted image 20260511095707.png](/images/Pasted%20image%2020260511095707.png)

解：在n-1条边把所有的节点都联通，且不可生成环路这一前提下，依次找出最短的边（图中值从小到大依次为12345），即先把值为1的边都画出来，再把值为2的边画出来，再把值为3的边画出来，此时满足了使用n-1条边把所有的节点都联调，且未生成环路，结果即为得到的所有边相加，得出答案为13。
